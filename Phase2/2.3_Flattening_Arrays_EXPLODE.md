# Flattening Arrays (EXPLODE)

> [!TIP]
> **「カラムの中にカンマ区切りで入っているリストを開きたい」**
> 昨今のデータ基盤（特にData LakeやNoSQL由来のデータ）において、データは綺麗な表形式（RDB）で連携されるとは限りません。
> JSONの配列（Array）を縦の行に展開する（Flattening）強力な技術をマスターしましょう。

---

## 1. ビジネスシナリオ：入れ子になったデータの展開

「ECサイトの注文システムから送られてくる購入データで、複数の購入アイテムが1つの配列（Array型）として1カラムに入ってしまっている。
これを商品ごとの売上ランキングとして集計したいので、**1行1アイテムになるよう縦に分解**したい。」

**【インプットデータ：注文テーブル（orders）】**

| order_id | user_id | items_array |
|---|---|---|
| O-001 | U-A | ["Apple", "Banana"] |
| O-002 | U-B | ["Orange"] |
| O-003 | U-C | [] (空配列) |

これを、以下のようにフラット化（展開）しなければBIツールで表現できません。

**【理想の出力】**

| order_id | user_id | item_name |
|---|---|---|
| O-001 | U-A | Apple |
| O-001 | U-A | Banana |
| O-002 | U-B | Orange |

---

## 2. 解決策：EXPLODE による縦への爆破

`EXPLODE` は、配列の中身を一つ一つ「爆発」させて、要素数と同じ数だけ行を増殖（複製）させてくれる関数です。

### 2.1 ✅ 基本構文（LATERAL VIEW）
Hiveなど、古くから使われている確実な記法です。

```sql
SELECT
  order_id,
  user_id,
  item_name
FROM orders
-- LATERAL VIEW EXPLODE を使用して、配列カラムを新しい仮想的なテーブルに変換し結合
LATERAL VIEW EXPLODE(items_array) AS item_name;
```

### 2.2 ✅ モダン構文（Databricks等）
Spark 3.x以降や最新のSQLエンジンでは、`SELECT` の中に直接関数のように記述できるようになり、圧倒的に短く書けます。

```sql
SELECT
  order_id,
  user_id,
  -- カラム内で直接爆破して別名をつける
  EXPLODE(items_array) AS item_name
FROM orders;
```

## 3. 実務の壁：EXPLODE の罠

`EXPLODE` は非常に強力ですが、実務で使うと必ずぶつかる2つの壁があります。

### 3.1 罠①：空配列のレコードが消滅する
`EXPLODE( [空配列] )` を実行すると、その親行（上記の O-003 の注文レコード）自体が**結果から完全に消滅します**（INNER JOINのような扱いになるため）。
「買った商品は無いけど、注文ログとしては残したい」場合に集計が狂います。

**【解決策】：EXPLODE_OUTER**
`LEFT JOIN` の性質を持つ `EXPLODE_OUTER` を使います。

```sql
SELECT order_id, EXPLODE_OUTER(items_array) AS item_name FROM orders;
-- 結果： O-003 | NULL  として行は担保される
```

### 3.2 罠②：順番が分からなくなる（元は何番目だった？）
例えば、「一番目にカートに入れた商品と、最後にカートに入れた商品を分析したい」場合、単純な `EXPLODE` では配列の元のインデックス（インデント順位）が失われます。

**【解決策】：POSEXPLODE（Position Explode）**
配列の「値」とともに、「インデックス番号（0始まり）」も同時に返してくれます。

```sql
SELECT
  order_id,
  pos AS array_index, -- 0, 1, 2...
  item_name
FROM orders
LATERAL VIEW POSEXPLODE(items_array) AS pos, item_name;

-- 新構文の場合（結果が配列などで返る）
-- SELECT order_id, POSEXPLODE(items_array) FROM orders;
```

## 4. 確認クイズ（理解度チェック）

### Q1. カンマ区切りのただの文字列 `"Apple,Banana"` が入ったカラムを EXPLODE するには？

<details>
<summary>💡 回答を見る</summary>

直接文字列を変数に入れることはできません。まず `SPLIT` 関数を使って文字列を強引に Array（配列）に変換してから爆破します。

```sql
SELECT EXPLODE(SPLIT(item_string, ',')) as item_name FROM orders;
```
</details>

### Q2. ログ内の1行の中で2つの異なる配列（例: views_array と clicks_array）を同時に EXPLODE した場合はどうなる？

<details>
<summary>💡 回答を見る</summary>

**CROSS JOIN爆発が起きます。**
viewsが3要素、clicksが3要素ある行の場合、1回の `SELECT` 区句内で2つ `EXPLODE` すると、3 × 3 = 9行に増殖してしまい、全く意味のない組み合わせが生成されます。
複数展開が必要な場合は、CTEを分けて段階的に処理するか、配列同士をZIP（結合）してから展開する必要があります。
</details>
